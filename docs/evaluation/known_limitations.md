# 当前系统边界（Known Limitations）

本页从评估角度汇总当前系统的主要局限，便于设置正确的使用预期。

## 覆盖不足的场景

- 强语境 BEC：缺少通讯录/历史线程/付款流程上下文
- 高质量 APT：文案自然、技术信号弱，规则式语义召回有限

## 取证能力边界

- URL 不做真实跳转与落地页分析（默认离线）
- 附件不解包、不做内容静态/动态分析
- Header 解析为简化版本（可能漏掉复杂变体）

## 风险管理建议

- 将 `suspicious` 作为“需要更多上下文/人工复核”的主要出口
- 在集成层补充组织上下文证据（allowlist、联系人、线程、财务系统）
- 对扩展的在线工具启用 sandbox 与配额/超时控制

更细粒度的失败模式见 `engine/failure_modes.md`。
